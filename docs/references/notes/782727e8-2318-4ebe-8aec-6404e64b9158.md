# Scaffold Rust project with dependencies

Initialize the Cargo project and configure all dependencies needed for the TUI app.

## Scope
- `Cargo.toml` with dependencies: `ratatui`, `crossterm`, `octocrab`, `tokio`, `anyhow`, `clap`
- `src/main.rs` with basic async main and terminal setup/teardown
- Update `.gitignore` to include Rust-specific entries (`/target`)
- Remove the default JS-oriented `.gitignore` entries

- [Spec](intent://local/note/spec)

- App accepts `--owner` and `--repo` CLI args (or positional `owner/repo`)


## Acceptance Criteria

- ✅ `cargo check` passes with no errors
- ✅ Terminal initializes and restores cleanly on exit (raw mode + alternate screen)
- ✅ App accepts positional `owner/repo` CLI arg via clap
- ✅ `cargo run -- --help` shows usage

## Changes

- `Cargo.toml` — created via `cargo init`, configured with ratatui 0.29, crossterm 0.28, octocrab 0.41, tokio 1, anyhow 1, clap 4
- `src/main.rs` — async main with clap CLI parsing (`owner/repo` positional arg), terminal init/teardown (raw mode + alternate screen), basic event loop using `app` and `event` modules
- `src/app.rs` — pre-populated by hook with App state, Issue struct, View/StateFilter enums
- `src/event.rs` — pre-populated by hook with keyboard event handling
- `src/github.rs` — pre-populated by hook with octocrab client
- `.gitignore` — replaced JS-oriented entries with Rust entries (`/target`, `Cargo.lock`)

## Learnings

- A hook auto-generates `pub mod` declarations and stub module files when `src/main.rs` is written. The `pub mod event;` declaration conflicts with `crossterm::event` imports, so crossterm event types must be referenced via fully qualified paths or the local `event` module must be used instead.
