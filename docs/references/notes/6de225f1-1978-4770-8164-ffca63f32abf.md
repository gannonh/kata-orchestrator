# Build app state and event handling

Create the application state model and keyboard event loop.

## Scope
- `src/app.rs` — App struct holding state (issue list, selected index, current view, loading state, input mode)
- `src/event.rs` — Event loop using crossterm reading keyboard events and forwarding to app
- Views: `IssueList`, `IssueDetail`
- Key bindings: `j/k` or arrows to navigate, `Enter` to view detail, `Esc`/`q` to go back/quit, `o`/`c` to filter open/closed, `r` to refresh

## Learnings
