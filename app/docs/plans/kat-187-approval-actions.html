Type Model
message-decision-parser.ts
type InlineDecisionActionId = | 'approve_tech_stack_plan' | 'keep_last_stack_switch' | 'ask_for_clarification' type InlineDecisionAction = { id: InlineDecisionActionId label: string followUpPrompt: string // canonical text sent via submitPrompt variant: 'default' | 'secondary' | 'outline' } type InlineDecisionCard = { sourceMessageId: string // ties card to originating agent message promptLabel: string actions: InlineDecisionAction[] }
Detection contract: Only agent messages eligible. Must contain "Approve this plan" + "Approve the plan" + "Keep the last switch" patterns. Unknown shapes fail closed.
Component Architecture

ChatPanel Modified

Extracts InlineDecisionCard from eligible agent messages. Derives decision state (available / pending / resolved) from runState and message history. Routes action clicks to submitPrompt(followUpPrompt).

MessageBubble Modified

Receives optional decisionCard, decisionState, and onDecisionAction props. Renders action row below markdown content when a card is present. Shows "Decision sent" label when resolved.

MessageActionRow New

Generic reusable action button row. Receives actions[], disabled, onAction. Uses shadcn Button primitives with variant support.

Supporting modules
  • message-decision-parser.ts New Exports extractInlineDecisionCard() and isDecisionResolved()
  • useIpcSessionConversation Existing Provides messages, runState, and submitPrompt. No changes needed.

    P-->>CP: boolean
    CP->>MB: message + card + state + callback

    Note over AR: User clicks Approve
    AR->>CP: onAction with actionId
    CP->>RT: submitPrompt with followUpPrompt
    RT-->>H: New user message appended
    Note over H,RT: Decision now resolved via message ordering
Resolution Semantics
STEP 1

Click Action

Action row disabled. submitPrompt called with canonical follow-up text.

→
STEP 2

Run Executes

Follow-up prompt persisted as normal user message. Agent responds.

→
STEP 3

Resolved

isDecisionResolved detects matching user message after source. Card renders as non-interactive.

Replay-safe: No additional persistence fields required. Resolution derives from message ordering in the existing conversation history. Works identically on replay.
