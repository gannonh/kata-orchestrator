# Space Creation Refinements Design

**Issue:** KAT-156
**Branch:** `feature/kat-156-space-creation-refinements`
**Approach:** Progressive Disclosure Refactor

## Decisions

| Decision | Choice |
|----------|--------|
| Auto-name format | Nanoid-style: `{repoLabel}-{nanoid(4)}` (e.g. `kata-cloud-x7k2`) |
| Browse validation | Require `.git` directory in selected path |
| GitHub repo source | `gh` CLI (`gh repo list`), auto-populated + searchable |
| PR support in branch picker | Branches only (PRs deferred) |
| Right panel grouping | Always group by repo with visible headers; fix current visual hierarchy |
| Existing folder flow | Shares `RepoPathPicker` component with copy-local flow |

## Form Structure

3 steps with progressive reveal. Old Step 3 (manual naming) eliminated.

### Step 1: Where work happens (unchanged)

Two buttons: "Managed workspace (worktree)" | "Existing folder/worktree"

### Step 2: Source setup (restructured)

Each flow starts minimal and reveals fields as the user progresses.

**Copy local repo:**
- Initial: read-only path input + Browse button
- Browse opens native Electron `dialog.showOpenDialog` (directory mode)
- On selection: validate `.git` exists. If valid, Branch picklist appears (populated from `git branch --list`). If invalid, inline error.

**Clone from GitHub:**
- Initial: searchable combobox pre-populated with user's repos (via `gh repo list`)
- On repo selection: Branch picklist appears (populated from `gh api repos/{owner}/{repo}/branches`)

**Create new repo:**
- Two inputs: "New repo parent directory" and "New repo name"
- No browse button or branch picker needed

**Existing folder/worktree:**
- Same `RepoPathPicker` component as copy-local: path input + Browse + validate `.git` + Branch picklist

### Step 3: Review and create (was Step 4)

- Auto-generated name shown as summary line: `Space: kata-cloud-x7k2`
- Other summary lines (workspace mode, source action, path)
- Create button

No manual space name input.

## Component Architecture

### New shared components

**`RepoPathPicker`** (copy-local + existing folder)
- Props: `path`, `onBrowse`, `branches`, `selectedBranch`, `onBranchChange`, `isLoadingBranches`, `error`
- Renders: read-only input + Browse button + conditional branch picklist
- Browse calls IPC `dialog:openDirectory` (Electron `dialog.showOpenDialog`)
- Parent calls IPC `git:listBranches` after valid path selection

**`GitHubRepoPicker`** (clone from GitHub)
- Props: `repos`, `selectedRepo`, `onRepoSelect`, `isLoadingRepos`, `searchQuery`, `onSearchChange`, `branches`, `selectedBranch`, `onBranchChange`, `isLoadingBranches`
- Renders: searchable combobox + conditional branch picklist
- Repos fetched via IPC `github:listRepos`
- Branches fetched via IPC `github:listBranches`

### Modified components

**`CreateSpacePanel`** ‚Äî Reduced prop surface. Receives:
- Mode/method selection (unchanged)
- `repoPath` + `branches` + `selectedBranch` (local flows)
- `githubRepos` + `selectedRepo` + `repoBranches` + `selectedBranch` (GitHub flow)
- `newRepoParentDir` + `newRepoFolderName` (new repo flow)
- `autoGeneratedName` (read-only)
- Loading/error states
- Callbacks

**`HomeSpacesScreen`** ‚Äî Owns new IPC calls and state for branches/repos.

**`SpacesListPanel`** ‚Äî Fix visual hierarchy: repo group header must be visually distinct from space item rows (larger/bolder text, spacing).

### New IPC channels

| Channel | Input | Output |
|---------|-------|--------|
| `dialog:openDirectory` | none | `string \| null` (selected path) |
| `git:listBranches(repoPath)` | repo path | `string[]` branch names |
| `github:listRepos` | none | `Array<{ name, nameWithOwner, url }>` |
| `github:listBranches(owner, repo)` | owner, repo name | `string[]` branch names |

## Auto-Naming Algorithm

Replace `resolveSpaceName` in `space-name.ts`:

1. Extract `repoLabel` from source (last path segment for local, repo name for GitHub, folder name for new repo)
2. Generate 4-character nanoid (lowercase alphanumeric)
3. Compose: `{repoLabel}-{nanoid}` (e.g. `kata-cloud-x7k2`)
4. Check against existing names; regenerate on collision (cap at 10 attempts, then use 6-char)
5. No user override

### Type changes

**`CreateSpaceInput`:** Remove `spaceNameOverride` and `name` fields. Name always derived server-side.

**`SpaceRecord`:** No changes. `name` stores the auto-generated value.

### Field relabeling (Create new repo)

- "Source repo parent directory" -> "New repo parent directory"
- "Source repo folder name" -> "New repo name"

## Error Handling

| Scenario | Behavior |
|----------|----------|
| Browse selects non-git directory | Inline error: "Selected directory is not a git repository." Path stays visible. Browse remains clickable. |
| `git branch --list` fails | Inline error: "Could not read branches." Fall back to text input for manual entry. |
| `gh` not installed/authenticated | Inline error: "GitHub CLI not available. Install and authenticate with `gh auth login`." |
| `gh repo list` fails | Error in combobox area. "Or paste URL" fallback link reveals text input. |
| Empty branch list | Picker shows "No branches found", defaults to `main`. |
| Nanoid collision | Regenerate (loop cap 10, then 6-char). Practically unreachable. |
| Space creation failure | Existing red error text in review step. |
