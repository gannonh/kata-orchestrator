03 Left Sidebar Structure
Sessions
Disclosure triangle header. Session name labels, active session highlighted. Click switches activeSessionId. Expanded in Mock 10, collapsed in Mocks 12-14.
Agents
Disclosure triangle header. Agent name + colored avatar dot (8-10px circle). "Kata Agents" (system) and "MVP Planning Coordinator" (session). No agent identity entity in current data model.
Tasks
Checkbox + title text. Checked = complete. Derived from spec SpecTaskRecord. Maps status: 'complete' to checked state. Content evolves from process items ("Spec submitted") to implementation items ("Bootstrap desktop shell").
Conversation Entries
Bottom of sidebar in Mock 10 only. Short label + timestamp ("Spec Updated", "I updated the..."). Click scrolls center panel. Not visible in Mocks 12-14.
Expand/Collapse Behavior
Each section has a disclosure triangle to the left of its heading. Collapsed: triangle points right, content hidden. Expanded: triangle points down, content visible. Mock 10 shows all expanded. Mocks 12-14 show Sessions and Agents collapsed, Tasks expanded. State is local UI only, not persisted.

Critical Path: New Types Needed
Two new types block most features:

1. ConversationMessage - Required by: conversation thread, conversation entry index, conversation persistence, approval buttons, message input. Fields: id, role, agentName, content (markdown), timestamp, actionButtons (optional).

2. AgentRecord - Required by: agent roster in sidebar. Fields: id, name, avatarColor, role. Currently only string labels exist in OrchestratorDelegatedTaskRecord.specialist.
